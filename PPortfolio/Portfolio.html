Description

This is my second website, created after taking a break from coding and wanting to continue improving my skills in web development. I decided to build my own personal portfolio as a place to showcase my projects, present myself and make it easier for others to see my work and contact me.

The purpose of this website is to serve as my professional online presence as I start exploring freelance opportunities. It is a platform I plan to keep updating and expanding as I continue learning, building new projects and growing as a web developer.

Technologies / Creation Process

This project was developed using HTML, CSS, SVG and basic JavaScript (for language switching functionality). After a longer break from coding, I wanted to fully refresh and improve my frontend skills while building something meaningful and personal.

The initial idea was to create a simple portfolio that I could use on platforms like GitHub or freelance websites. However, the concept quickly evolved into a more complex personal website where I could present myself, my projects and future work in one place.

I started by sketching design ideas on paper, planning the layout and writing content for each section. Then I carefully built the website step by step — tag by tag — constantly revising, improving and fixing issues as I discovered them.

What I Learned

Building my personal portfolio taught me a wide range of new web development skills. I worked with modern frontend techniques such as CSS Grid for layout structure, aspect-ratio for proper image scaling and improved semantic HTML for a more accessible and organized page structure.

Key Takeaways

  • Using favicons and understanding how browsers handle them
  • Basic SEO (meta tags, page titles, keywords)
  • CSS Grid, Flexbox and more advanced styling techniques
  • Maintaining correct aspect ratios for web images
  • Better handling of file paths, linking and project structure
  • Basic JavaScript for dynamic content (language switcher)

This was my first fully self-directed project — no predefined structure or design requirements. It taught me how to plan, design and build a web application from scratch while making many decisions independently and continuously improving the final result.
